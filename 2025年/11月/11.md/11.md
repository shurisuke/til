

# インフラとは
- インフラ = インフラストラクチャー（生活や産業の基盤）
- IT分野ではアプリケーションを支える 基盤 のこと
- ハードウェア：サーバー、ストレージ、ネットワーク機器
- ソフトウェア：OS、各種サーバー（Web/AP/DB）


## インフラ構築時の注意点
- セキュリティ：不正アクセス防止のため設定が重要
- 可用性：常にサービスが稼働する状態を維持
- 冗長性：1台停止しても他でカバーできる構成（例: サーバー2台）


サーバー基礎

## サーバーとは？
- サーバー = サービスを提供するプログラムまたはコンピューター
- クライアント = リクエストを送る側（例: Webブラウザ）

## サーバーの種類と例
- Webサーバー：Apache, Nginx
- APサーバー：Unicorn, Tomcat
- DBサーバー：MySQL, PostgreSQL
- メールサーバー：Postfix

## APサーバーの処理の流れ（Rails）
- ブラウザ → Webサーバーへリクエスト
- Webサーバー → APサーバーへ転送（動的処理）
- APサーバー → DBサーバーからデータ取得
- データをもとにHTMLを生成しWebサーバーへ返却
- ユーザーへレスポンスを返す

データベースサーバー基礎

## DBサーバーとは
- DBを管理する DBMS を動かすサーバーのこと
- 高速処理、トランザクション、耐障害性が特徴

## RDBの主な種類
- MySQL / PostgreSQL / Oracle / SQL Server / SQLite

## APサーバーとの接続情報（例: database.yml）
- ユーザー名
- パスワード
- ホスト（IPまたはドメイン）
- ポート番号（MySQL: 3306など）
- データベース名


ネットワーク基礎

## IPアドレスとは
- コンピューターの 住所
- 種類：
- グローバルIP：インターネット用（ルーターが所持）
- プライベートIP：LAN内用（自分のPCなどが所持）

## IP確認コマンド
- Mac：
- networksetup -listallhardwareports
- ifconfig en0
- WSL/Ubuntu：
- ip address show
- ip address show eth0

## IPアドレスの形式
- サブネットマスク：ネットワーク部とホスト部を区別
- CIDR表記：例 192.168.10.104/24

DNSと名前解決

## DNSとは？
- ドメイン名 → IPアドレスに変換する仕組み
- 名前解決には nslookup コマンドが使える
- 例：nslookup github.com
