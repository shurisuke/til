# jQueryとSwiperについて
- jQueryはJavaScriptを簡潔に書くためのライブラリで、DOM操作やイベント処理、Ajax通信などを簡単に実装できる。
- Swiperは画像スライダーを実装するための有名なライブラリで、スマホ対応・タッチ操作に強く、カスタマイズ性も高い。


# Swiperの導入方法（基本）
- Swiperは画像やコンテンツをスライド表示できる人気のJavaScriptライブラリ。
- モバイル対応・レスポンシブ・タッチ操作が標準で使える。

## CDNから導入する方法
- CSSとJSをそれぞれCDNで読み込む。
- HTML構造を用意する
- JavaScriptでSwiperを初期化する

CDNではなくnpmで導入する場合はnpm install swiperを実行。
Railsで使う場合はWebpackやimport map、アセットパイプラインなど環境に応じて読み込み方法が異なる。
CSSとJavaScriptを読み込んだだけでは機能しないため、HTML構造と初期化処理が必須。


# Rubyの %i と %w について
%i
- %iはシンボルの配列を作るための記法。
- 各要素は :シンボル として扱われる。
- クォートやカンマなしで簡潔に複数のシンボルを定義できる。

%w
- %wは文字列の配列を作るための記法。
- クォートやカンマを使わずに、スペースで区切って複数の文字列を定義できる。


# missing required keys: [:id] エラーについて

## このエラーが出る原因
*_path や *_url のようなルーティングヘルパーを使う際に、必要なキー（通常は:id）を渡していない場合に発生する。

典型的な例
edit_article_pathのように書いてしまうと、id パラメータが不足しているため、
ArgumentError: missing required keys: [:id] というエラーになる。
正しくはedit_article_path(@article)のように必要なキーを持つオブジェクト or idを指定して渡す

ネストされたルーティングの場合は、親リソースも含めたパラメータが必要になる。
edit_admin_article_path(admin_id: 1, id: 2)
